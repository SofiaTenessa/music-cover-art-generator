## AI Development Tools

### Claude (Anthropic) via Cowork and Claude Code

**Tools used:** Claude Cowork mode (initial project scaffolding) and Claude Code CLI (implementation, debugging, iteration).

**What was AI-generated:**

- *Initial project scaffold*: Directory structure, `requirements.txt`, `README.md`, `SETUP.md`, and this `ATTRIBUTION.md` template were generated by Claude based on a detailed project plan.
- *Starter code*: The initial versions of `src/preprocessing.py`, `src/dataset.py`, `src/model.py`, `src/train.py`, `src/evaluate.py`, `src/prompt_builder.py`, `src/pipeline.py`, `src/baselines.py`, and `app/streamlit_app.py` were scaffolded by Claude with TODO comments identifying decisions for the author.

**What I modified / authored myself:**

*Fill this in as you go. Be specific. Examples:*

- Designed the final CNN architecture — chose kernel sizes, depth, and dropout rates based on ablation experiments in `docs/ablation_results.md`.
- Wrote the prompt construction logic in `src/prompt_builder.py` — the mapping from genre + mood features to descriptive text prompts is my own design, tuned by qualitative evaluation of ~30 generations.
- Implemented SpecAugment in `src/dataset.py` from scratch after reviewing the original paper (Park et al., 2019).
- Tuned hyperparameters (learning rate, batch size, SpecAugment mask widths) across 12 configurations — see `docs/hyperparameter_search.md`.
- Implemented the error analysis and confusion-matrix visualization in `src/evaluate.py`.

**What I had to debug / rework:**

*Fill in as you encounter issues. Examples:*

- The initial MPS device detection in `src/train.py` failed silently on macOS — I discovered PyTorch's `amp` context doesn't support MPS and added a CPU/CUDA fallback.
- Claude's first pass at the Stable Diffusion integration used deprecated `StableDiffusionPipeline.from_pretrained` arguments; I updated to the current diffusers API and added torch.float16 handling.
- The confusion matrix plotting code mismatched the label ordering; fixed in `src/evaluate.py`.
